Executive Summary
LYRN is a fully local cognitive architecture designed for long-running autonomous agents that operate without cloud services, external alignment layers, or proprietary memory systems. The system uses structured text snapshots, persistent verbatim episodic memory, specialized job pipelines, and a delta manifest that acts as a real-time awareness layer.
01 // Introduction

Traditional chatbot-style interfaces are not designed for long-running autonomous agents. They lose context, rely on external servers, and depend heavily on safety layers or network-based memory systems.

LYRN approaches cognition from a different angle. Instead of remote APIs, LYRN uses a transparent local-first method based on structured snapshot files, a minimal delta manifest, and a job-driven control loop.

LYRN is driven by three core ideas:

  • All intelligence should be controlled by the user through transparent text.
  • Memory should be persistent and verifiable.
  • Autonomy should be modular and driven by explicit job instructions.
02 // Current Capabilities

Automation Dashboard

The dashboard allows direct IPC pipelines between Python scripts and the LLM process. Jobs are treated as self-contained text blocks with trigger conditions. When a snapshot is modified, the LLM reprocesses the updated text from that point forward, providing instant configurability without losing state.

Verbatim Episodic Memory

LYRN stores all conversation pairs as plain text files. Every fifty pairs are grouped into a block with a summary. Files are small (3-5KB), meaning years of operation result in only 1-2GB of data. This avoids "vector drift" and keeps a perfect historical record.

03 // System Architecture

Snapshot Layer

A structured text file defining identity. It includes System Instructions, Rules, Personality Parameters, AI Preferences, and Job Instructions. It acts as the static cognitive scaffold.

Delta Manifest

A single text file representing "current awareness." It holds timers, system metrics, and simulated sensor inputs. The LLM only reads this when performing a job, minimizing token usage.

04 // Cognitive Loop
THE LOOP:
1. Load snapshot
2. Load delta manifest
3. Receive job triggers or user input
4. Execute job instructions
5. Perform LLM run
6. Update memory and manifest
